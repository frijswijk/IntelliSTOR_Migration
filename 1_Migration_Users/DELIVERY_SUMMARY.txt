FEATURES
================================================================================

Both implementations support:
  ✓ Read three permission CSV files from any folder
  ✓ Parse CSV format with proper header detection
  ✓ Extract RID column values
  ✓ Handle pipe-delimited RIDs (split and deduplicate)
  ✓ Automatic deduplication using set data structure
  ✓ Numeric sorting of RIDs
  ✓ Generate CSV output with standard format
  ✓ Custom output filename support
  ✓ Progress reporting during processing
  ✓ Comprehensive error handling
  ✓ Exit codes for automation
  ✓ Identical output from both implementations

INPUT FILES REQUIRED
================================================================================

Each folder must contain exactly these three CSV files:
  1. STYPE_FOLDER_ACCESS.csv     (folder permissions)
  2. STYPE_REPORT_SPECIES_ACCESS.csv (report species permissions)
  3. STYPE_SECTION_ACCESS.csv    (section permissions)

Each CSV file must contain a "RID" column.

LIMITATIONS & NOTES
================================================================================

Known Limitations:
  - Requires exact CSV file names (case-sensitive on Linux/macOS)
  - Requires exactly three CSV files (no more, no less)
  - RID column must be named exactly "RID"
  - Input files must be UTF-8 encoded

Notes:
  - Pipe-delimited RIDs are automatically handled
  - Empty RID fields are skipped
  - Whitespace is trimmed automatically
  - Numeric sorting handles all RID formats
  - Large files (1M+ rows) work fine but take longer
